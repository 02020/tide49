# VIA → Tide49 定制化改造记录

## 一、项目背景

[VIA](https://usevia.app/) 是一个键盘配置 Web 应用，通过 WebHID 协议连接物理键盘，在浏览器中修改键位映射、灯光等。本改造将其从通用键盘配置工具定制为 **Tide49 分体键盘的专用配置页面**，核心目标：

- 移除所有 USB 设备搜索/授权流程
- 启动即加载 Tide49 键盘定义（4×14 矩阵布局）
- 6 个图层全部展开，无需层切换操作
- 2 列网格布局，每行两个键盘

## 三、详细修改说明

### 3.1 入口层：跳过 HID 检测

**文件**：`src/Routes.tsx`

```typescript
// 修改前：动态检测浏览器 HID 支持
const hasHIDSupport = 'hid' in navigator;

// 修改后：硬编码为 true，跳过所有 HID 检查
const hasHIDSupport = true;
```

### 3.2 主页组件：移除设备搜索流程

**文件**：`src/components/Home.tsx`

原代码包含：
1. USB 授权弹窗提示
2. 设备连接状态轮询（`reloadConnectedDevices`）
3. USB 错误处理页面
4. 仅连接时显示子路由

现代码简化为：
1. 挂载时派发 `initTide49()` 
2. 定义加载后清除选中键状态
3. 直接渲染子路由

### 3.3 Tide49 键盘定义

**文件**：`src/utils/tide49-definition.ts`（新建）

核心常量：

| 常量 | 值 | 说明 |
|------|-----|------|
| `TIDE49_DEFINITION` | `VIADefinitionV3` | 经 `@the-via/reader` 转换的完整键盘定义 |
| `TIDE49_VENDOR_PRODUCT_ID` | `0x342D * 65536 + 0xE47A` | Epomaker 硬件 ID |
| `TIDE49_PATH` | `'tide49-local'` | Mock 设备路径 |
| `TIDE49_LAYER_COUNT` | `6` | 图层数 |
| `TIDE49_KEY_COUNT` | 从定义动态计算 | 按键数 |

布局来源：`docs/Epomaker_tide49.json`（4 行 × 14 列矩阵，含完整 KLE 键位布局）。

处理流程：原始 KLE JSON → `keyboardDefinitionV3ToVIADefinitionV3()` → 转换为 VIADefinitionV3 格式（包含 `layouts.keys` VIAKey[] 数组、`menus` 菜单配置、`customKeycodes` 自定义键码等）。

### 3.4 初始化 Thunk

**文件**：`src/store/devicesThunks.ts` → `initTide49()`

执行顺序：
1. `updateDefinitions` — 注入键盘定义到 Redux store
2. `updateSupportedIds` — 注册 Tide49 为受支持设备
3. `updateConnectedDevices` — 创建 mock 已连接设备
4. `selectDevice` — 选中 mock 设备
5. `setNumberOfLayers` — 设置 6 层（使用原始 action 格式 `{type: 'keymap/setNumberOfLayers', payload: 6}`）
6. `loadLayerSuccess` × 6 — 预填充 6 层空键位表

### 3.5 Mock 设备不创建真实 HID

**文件**：`src/store/devicesSlice.ts` → `getSelectedKeyboardAPI`

```typescript
export const getSelectedKeyboardAPI = createSelector(
  getSelectedDevicePath,
  (path) => {
    if (!path || path === TIDE49_PATH) return null;  // mock 路径返回 null
    return new KeyboardAPI(path);
  },
);
```

关键修复：绕过 `new KeyboardAPI(path)` 中尝试创建真实 HID2 连接导致的 `Missing hid device in cache` 运行时错误。

### 3.6 ConfigureKeyboard 扩展

**文件**：`src/components/n-links/keyboard/configure.tsx`

新增两个可选 prop：
- `matrixKeycodes?: number[]` — 允许父组件直接传入键位数据，绕过 store 中的 `getSelectedKeymap`
- `layerLabel?: string` — 图层标签，当前未使用但保留扩展

### 3.7 2D 模式：6 层全展开网格

**文件**：`src/components/two-string/canvas-router.tsx`

**布局结构**：
- 外层容器：`flex: 1; minHeight: 0`（在 flex column 中占满剩余空间）
- `AllLayersContainer`：`position: absolute; width: 100vw; height: 100%` + 可滚动
- `AllLayersGrid`：CSS Grid `grid-template-columns: 1fr 1fr; gap: 50px 40px;`
- `LayerCell`：圆角边框卡片，`padding: 20px 16px 16px; min-height: 200px`

**数据流**：
- `AllLayersGridView` 通过 `useAppSelector(getSelectedKeymaps)` 获取全部 6 层键位数据
- 每层对应一个 `LayerCell`，内嵌 `ConfigureKeyboard`
- `ConfigureKeyboard` 通过 prop `matrixKeycodes={layer.keymap}` 接收该层数据，不给 store 的 `getSelectedKeymap`

**路由选择**：
- `KeyboardGroup` 组件检测 `path === '/'` 时，渲染 `AllLayersContainer` 替代原始的单层切换
- 非 configure 路由仍保留原始滑动导航（Test/Design）

**CSS 间距（UI 优化）**：
- 网格行间距 50px，列间距 40px
- 外层 padding: 24px 32px
- 卡片内边距 20px/16px，边框 2px，圆角 12px
- Layer 标签：`font-size: 14px; font-weight: 700; letter-spacing: 0.5px`

### 3.8 3D 模式：6 层 3D 网格

**文件**：`src/components/three-fiber/canvas-router.tsx`

`AllLayers3DGroup`：Three.js 场景中的 2 列 × 3 行布局
- 列间距 22 单位，行间距 15 单位
- 每层用 `<Html>` 组件显示 Layer 标签
- 容器高度固定为 500px，`flex: 1`
- 路由为 `/` 时渲染 `AllLayers3DGroup`，否则渲染原始 `KeyboardGroup`

### 3.9 Pane 层简化

**文件**：`src/components/panes/configure.tsx`

移除内容：
- `Loader` / `ChippyLoader` / `Authorize device` 按钮
- `LayerControl` 组件（层切换 UI）
- 对应的 import 语句

保留内容：
- ConfigurePane：等待定义和层数据就绪后直接渲染 `ConfigureBasePane`
- ConfigureGrid：侧边菜单 + 功能面板
- `Badge` 组件（顶部信息条）

### 3.10 React key prop 警告修复

**文件**：`src/components/two-string/key-group.tsx`

问题：`getKeycapSharedProps()` 返回对象中包含 `key` 属性，通过 `{...props}` spread 传递给 `<Keycap>`，React 警告 `key` 必须在 JSX 中作为直接属性传递。

修复：从共享 props 中解构出 `key`，作为 `<Keycap key={keycapKey}>` 直接属性单独传递。
```tsx
const {key: keycapKey, ...restSharedProps} = sharedProps;
return <Keycap key={keycapKey} {...restSharedProps} />;
```

（3D 版本的 `key-group.tsx` 已正确处理此模式，无需修改。）

## 五、问题修复记录

### 5.1 运行时错误：`Missing hid device in cache`

- **症状**：页面白屏，控制台报 HID2 构造函数错误
- **原因**：`getSelectedKeyboardAPI` selector 对 mock 路径 `tide49-local` 创建真实 HID 连接
- **修复**：`devicesSlice.ts` 中加路径检查，返回 null

### 5.2 空白页面（高度坍缩）

- **症状**：页面加载后内容不可见
- **原因**：`#root` 是 flex column，容器 div 无显式高度，所有子元素 `position: absolute`，高度坍缩为 0
- **修复**：2D 模式加 `flex: 1; minHeight: 0`，3D 模式加 `flex: 1; height: 500`

### 5.3 布局核对错误

- **症状**：加载了 108 键的测试布局而非实际 Tide49 布局
- **原因**：最初使用 `test-keyboard-definition.json`（1×108 矩阵）
- **修复**：改用 `docs/Epomaker_tide49.json`（4×14 矩阵），通过 `@the-via/reader` 正确转换

### 5.4 React key prop 警告

- **症状**：控制台报 `A props object containing a "key" prop is being spread into JSX`
- **修复**：从 `getKeycapSharedProps` 返回值中解构 `key` 作为直接属性传递

### 5.5 层数错误

- **症状**：只显示 4 层而非 6 层
- **原因**：`AllLayersGridView` 中 `Array.from({length: 4})` 笔误
- **修复**：改为 `length: 6`
